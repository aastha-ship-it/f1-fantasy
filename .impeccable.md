# Design Context — F1 Fantasy

*(Source of truth for all design work. The `impeccable` skill reads this on every future design task. Established via `/impeccable teach` on 2026-04-19 and refined by `/plan-design-review`.)*

## Users

5-10 close friends with shared F1 fandom. Primary context: **Sunday evening on the couch during a Grand Prix, phone in hand, commercial-break attention span**. Secondary context: Friday qualifying, weeknight trash-talk check-ins. They know F1 deeply (driver numbers, team rivalries, circuit quirks) and they're playing against each other, not the house. Nobody is a "new user" being onboarded to F1 — they're here because they already argue about Leclerc vs Norris every weekend.

## Brand Personality

**Three words:** broadcast, sparse, ritualistic.

- **Broadcast** — the product borrows aesthetic energy from F1 TV graphics, NBA Top Shot card unveils, and Drive to Survive title sequences. Not dashboard, not admin panel. Television-grade composition. Bold type, meaningful silence, cinematic beats.
- **Sparse** — silent where competitors stuff data. One big number over four small ones. No "at-a-glance stat grid" of ornamental metrics. If an element doesn't earn its pixels, it's cut.
- **Ritualistic** — a weekend rhythm. Friday: predict. Saturday: anticipate. Sunday: reveal. Monday: trash-talk. The UI reinforces that rhythm through state-aware home screens and the reveal ceremony.

**Emotional targets:**
- Predict flow → *conviction* ("I called it first")
- Lock countdown → *tension* (clock pressure, stakes rising)
- Reveal → *theatre* (poker showdown, collective gasp)
- Leaderboard → *stakes* (season-long narrative, you are Xth and Y is catching up)

## Aesthetic Direction

**Theme: dark.** Sunday-evening couch viewing, phone screens in dim rooms, race weekends often extend past sunset. Dark is the honest choice for this context.

**Reference galaxy (captures the feel):**
- F1 TV broadcast overlays — sector timing in tabular numerals, positional flip animations, team-color edge lighting
- NBA Top Shot pack openings — card-reveal cinematography, metallic surface treatments on hero moments, silence-then-drop pacing
- Drive to Survive title sequence — aggressive italic display type, high-contrast race-stripe compositions, cinematic color grading

**Anti-references:**
- Literal F1.com corporate-dashboard density (sponsor-heavy, eight-nav-items-at-once)
- Generic shadcn admin panel (rounded cards on gray grid, icon-above-heading repeated)
- Sports-betting neon flashiness (neon gradients, BET-NOW energy, manipulative color psychology)

## Typography

Three-font stack chosen to avoid the Inter/Plex/DM-Sans AI monoculture and deliver broadcast energy:

| Role | Face | Why | Source |
|---|---|---|---|
| **Display** | **Boldonse** (free) | Bold angular display sans with racing-stripe character. Used for event names ("MIAMI GP"), P1/P2/P3 callouts, countdown hero, reveal cards. | Google Fonts |
| **Body** | **Geist Sans** (free) | Modern grotesk, not in the reject list, web-optimized. Reads well down to 14px. Workhorse for running text, labels, form fields. | `next/font` or Vercel |
| **Tabular** | **Geist Mono** (free) | Monospace partner for scores, driver numbers, timers, leaderboard positions. Tabular figures prevent score jitter during state changes. Never used for generic "technical look" decoration. | `next/font` or Vercel |

**Rules:**
- Modular scale, ratio ~1.333 between steps. `clamp()` fluid sizing on marketing/hero; fixed `rem` on app chrome
- Line-height bumped +0.05-0.1 on dark surfaces (light type reads lighter weight, needs breathing room)
- Body text max-width `65-75ch`. No uppercase body — all-caps reserved for sub-labels and nav items
- Tabular figures (`font-variant-numeric: tabular-nums`) **mandatory** for: scores, timers, driver numbers, leaderboard positions, session countdowns

## Color Palette

Perceptually-uniform OKLCH throughout. All neutrals tinted toward the F1-red hue (~27° in OKLCH hue space) for subconscious cohesion.

```css
:root {
  /* Core surfaces — dark tinted warm-red */
  --bg:            oklch(16% 0.012 27);     /* near-black, faint red warmth */
  --surface:       oklch(21% 0.015 27);     /* card surfaces */
  --surface-2:     oklch(26% 0.018 27);     /* elevated surfaces, hover */
  --border:        oklch(34% 0.020 27);     /* subtle separators */

  /* Text — off-white tinted */
  --fg:            oklch(96% 0.008 27);     /* primary text */
  --fg-muted:      oklch(72% 0.015 27);     /* secondary text, labels */
  --fg-subtle:     oklch(52% 0.018 27);     /* tertiary, metadata */

  /* Brand accent — Ferrari-adjacent red, reserved for 10% use */
  --accent:        oklch(62% 0.22 27);      /* primary accent */
  --accent-hover:  oklch(68% 0.22 27);      /* interactive state */
  --accent-muted:  oklch(48% 0.15 27);      /* depressed/pressed */

  /* State colors — tinted toward brand hue */
  --success:       oklch(68% 0.18 150);     /* green for correct picks */
  --warning:       oklch(74% 0.18 70);      /* amber for T-60s lock warning */
  --error:         oklch(60% 0.22 15);      /* red-orange, not accent-red */

  /* Team color chips (used in driver cards, kept saturated) */
  --team-mclaren:   oklch(72% 0.17 55);     /* papaya */
  --team-ferrari:   oklch(58% 0.22 27);     /* red */
  --team-merc:      oklch(78% 0.08 180);    /* silver-cyan */
  --team-redbull:   oklch(38% 0.16 265);    /* deep blue */
  --team-aston:     oklch(50% 0.15 165);    /* racing green */
  --team-williams:  oklch(58% 0.14 245);    /* blue */
  /* ... add remaining teams as needed */
}
```

**60-30-10 visual weight:**
- 60% neutral surface (bg + surface + surface-2)
- 30% secondary text, borders, team-color hints
- 10% Ferrari-red accent — reserved for: lock-countdown pulse, LIVE-NOW indicator, perfect-podium badge, primary CTAs only. Overuse kills its power.

## Layout & Space

- **Spacing scale** (4pt-based, semantic names): `--space-xs: 4px, --space-sm: 8px, --space-md: 12px, --space-lg: 16px, --space-xl: 24px, --space-2xl: 32px, --space-3xl: 48px, --space-4xl: 64px`
- **Gap over margins.** No margin collapse hacks
- **Asymmetric compositions.** The reveal screen in particular should NOT be a centered 3-column grid — compose dramatically, offset cards, vary card sizes for hero picks
- **Container queries for cards** (`@container`), viewport queries for page-level layout
- **Self-adjusting grid** for leaderboard, driver picker: `grid-template-columns: repeat(auto-fit, minmax(280px, 1fr))`
- **Web-app posture.** Works on desktop AND phone via flex/grid. No mobile-first mandate. Touch targets ≥44×44px where tapped (submit buttons, driver picker, share button)

## Motion

- **Earned motion only.** Animations reserved for genuine state changes: submit confirmation, lock-countdown pulse, reveal card flip, leaderboard rank change
- **Exponential easing** (`cubic-bezier(0.22, 1, 0.36, 1)` — ease-out-quart) for natural deceleration. No bounce. No elastic
- **Transform + opacity only.** Never animate layout properties
- **`prefers-reduced-motion: reduce`** fully respected — reveal falls back to instant card display, lock pulse falls back to static state
- **Reveal choreography** (the product's emotional peak):
  1. Actual P1/P2/P3 result cards flip first, 300ms each, 200ms stagger
  2. Brief silence beat (~400ms)
  3. Friend pick cards flip in sequence, 150ms stagger, revealed alongside per-pick score breakdown
  4. Final leaderboard rank-change animation (subtle list reorder, no confetti)

## Lock Countdown Urgency (predict screen)

```
  T-24h to T-60s: normal state
    → countdown in --fg, neutral tone
    → submit button in --accent

  T-60s to T-5s: warning state
    → countdown color-shifts to --warning (amber, oklch(74% 0.18 70))
    → subtle 2Hz opacity pulse on countdown digits (respects prefers-reduced-motion)
    → banner appears below hero: "Predictions closing in 45s" in warning-amber
    → lock bar at bottom also shifts to warning-amber background
    → submit button stays accent (don't mix warning + red on same pixel)

  T-5s to T+: closed state
    → countdown reads "Locked" in --fg-muted
    → banner: "Predictions closed" in --fg-subtle
    → lock bar and submit button disabled, grayscale (--surface-2)
    → any submit attempt → "Predictions closed" toast, no API call
```

All animations use ease-out-quart easing.

## Team Identity Treatment (driver cards everywhere)

```
  Primary:    [●] MCL      (color dot in team color + 3-letter code, Geist Mono)
  Secondary:  small SVG team mark (optional, below primary, ~16×16px)
```

- Dot uses the `--team-{name}` OKLCH token
- 3-letter codes: MCL, FER, RBR, MER, AST, WIL, HAA, KIC, ALP, VRB (standard F1 team shorthand)
- **Team logos / marks:** optional enhancement. F1 team logos are trademarked; for the private friend league, acceptable risk. Ship with dot+code only if assets not ready; add SVG marks during implementation if you want the extra polish. If the product ever opens beyond friends, strip logos and keep dot+code clean.

## Interaction

- **Optimistic UI** for prediction submit — update button state immediately, reconcile with server
- **Progressive disclosure** — predict screen shows current event first, telemetry nudges below the fold
- **Empty states teach, don't apologize.** Pre-season empty leaderboard says "Season starts March 8. Here's last year's winner." not "No races yet"
- **No modals** unless truly unavoidable. Use inline drawers, sheets on mobile, full pages where content warrants
- **Primary button hierarchy strict.** One primary per view max. Rest are ghost / text-link / secondary

## Absolute Bans (AI-slop patterns)

NEVER write these — they are the most recognizable AI design tells:

1. **`border-left:` / `border-right:` with width > 1px on cards or list items.** The single most overused "design touch" in admin UIs. Use full borders, surface-lift, or no indicator at all.
2. **Gradient text** (`background-clip: text` with a gradient). Solid colors only for text.
3. Generic 3-column feature grid with icons-in-colored-circles
4. Purple/violet/indigo gradient backgrounds
5. Uniform bubbly border-radius on every element
6. Decorative blobs, floating circles, wavy SVG dividers
7. Emoji as structural design elements
8. Cookie-cutter section rhythm (hero → 3 features → testimonials → pricing → CTA)
9. Modals for every secondary action
10. Sparkline decorations

## Design Principles (priority order)

When in doubt, the higher-numbered principle loses.

1. **The reveal is the product.** Every screen serves the Sunday-night peak. Leaderboard, predict screen, countdown — all lead the user toward the next reveal moment. Cut anything that dilutes it.
2. **Sparse beats dense.** One big number > four small stats. Silence > filler. If an element doesn't pass the "would I miss this if it were cut?" test, cut it.
3. **Tabular everywhere numbers live.** Scores, timers, positions, driver numbers use monospace tabular figures. Numbers never jitter, shift, or reflow.
4. **Motion is earned, not decorative.** Only real state changes animate. No hover-glow, no decorative parallax, no "delight" spinners.
5. **Respect the couch-and-phone context.** Friends check during commercial breaks. Touch targets generous, primary actions reachable, key info scannable in 2 seconds.
