## Affected Pages/Routes

- `/join` — invite code gate; rejects invalid codes, redirects to `/login` on valid.
- `/login` — magic link request + verify flow via Supabase Auth.
- `/profile` — first-time setup; display name, fav team/driver fields.
- `/dashboard` — smart home card priority (predict-now if within 48h, reveal-waiting if results pending, else leaderboard).
- `/dashboard/predict` — prediction submission with 60s warning banner + 5s hard cutoff.
- `/dashboard/league` — leaderboard with streak badges.
- `/dashboard/standings` — driver + constructor standings (OpenF1 nightly pull).
- `/reveal/[eventId]` — cinematic card-flip reveal, `prefers-reduced-motion` fallback.
- `/api/share/[eventId]/card.png` — OG image route, publicly crawlable, revalidates hourly.
- `/admin/results/[eventId]` — admin-only manual result entry fallback.
- `/api/cron/fetch-results` — scheduled OpenF1 fetcher with retry + email alert.
- `/api/cron/sync-f1-data` — nightly calendar + driver reconciliation.

## Key Interactions to Verify

- **Invite gate:** valid code advances to login. Invalid code shows error, no auth path triggered.
- **Magic link signup:** email entered → link received → click → redirected to `/profile` (first time) or `/dashboard` (returning).
- **Prediction submit (race/quali):** select P1, P2, P3 from driver list → submit → confirmation with "editable until lock."
- **Prediction submit (sprint sessions):** only P1 required, P2/P3 fields absent.
- **Prediction edit before lock:** re-submit overwrites previous pick (UPSERT).
- **Lock boundary:** at T-60s warning banner appears. At T-5s submit disabled. At T-4s submission attempt → 409.
- **Reveal animation:** results land → reveal page shows result cards flipping, then friend picks flipping sequentially (~150ms stagger).
- **Reduced motion:** same reveal page with `prefers-reduced-motion: reduce` → final state instant, no flip animation.
- **Admin override:** non-admin hits `/admin/results/X` → 403. Admin submits results → scores compute → reveal page immediately reflects.
- **Share card:** reveal page share button → copies `/api/share/[eventId]/card.png` URL. URL returns valid 1200×630 PNG.
- **Streak badges:** leaderboard shows "Xn🔥 on pole picks" next to users with active P1 streaks.

## Edge Cases

- **DNF in prediction:** picked driver not in classified P1/P2/P3 → 0 pts for that slot, not counted as "wrong slot match."
- **Partial classification:** only 14 cars classified (rain-shortened race) → scoring still works for top 3 classified finishers.
- **Double-click submit:** UPSERT idempotent, no duplicate row.
- **Clock skew at lock:** server rejects even if client clock is ahead; DB trigger is final authority.
- **Mid-season calendar change:** race cancelled, OpenF1 meeting disappears → nightly sync removes event (or marks inactive); predictions for removed event remain but don't score.
- **Mid-season driver change:** new driver joins (e.g., reserve drivers) → nightly driver sync adds them to picker.
- **OpenF1 stale data:** fetcher sees same session but missing results → retries, after 2h email admin.
- **OpenF1 malformed JSON:** logged, skipped, admin alerted.
- **Empty leaderboard:** pre-season or no predictions yet → "no races yet" empty state.
- **Single char display_name:** accepted (no min length beyond 1).
- **Max length display_name:** enforced at 30 chars.
- **Unicode in display_name:** emoji allowed.
- **Share card with no results yet:** fallback PNG showing "results pending" (not a 404).
- **Invite code in URL vs form:** both paths supported; URL param `?code=XXX` pre-fills form.
- **Timezone edge:** user in GMT+14 (Kiribati) vs GMT-11 (Samoa) sees correct local lock time.
- **Mobile thumb zone:** submit button in lower 1/3 of mobile viewport.

## Critical Paths (must work end-to-end)

1. **Full signup → first prediction → reveal** (E1 + E3): join code → magic link → profile → predict → lock → results land → reveal animation plays.
2. **Score computation accuracy** (U1-U7): every scoring rule tested in isolation including DNF.
3. **OpenF1 failover** (I11 + I12 + I13): when OpenF1 fails, admin override path works, scores still compute, reveal still works.
4. **RLS security** (I6 + I7 + I8): nobody sees anyone's predictions before results, nobody self-escalates to admin.
5. **Lock boundary enforcement** (I2 + I3): predictions truly cannot be submitted after the bell, even with clock skew.
